Split Terminal 用户使用手册

版本：1.1.0 | 更新日期：2026-02-24

一、软件简介

Split Terminal 是一款基于 Electron 的多分屏终端管理器桌面应用，专为 Claude Code 用户设计。它允许你在一个窗口中同时管理多个终端会话，并深度集成了 Claude Code 的会话管理和定时任务调度功能。

核心特性

  • 多分屏终端：最多同时显示 6 个终端面板，支持多标签页管理
  • Claude Code 会话管理：浏览、搜索、恢复历史 Claude 会话，一键续接对话
  • 定时任务调度：定时自动执行 Claude 指令，支持每日定时、固定间隔、一次性执行
  • 文件浏览器：内置树形文件管理器，支持拖拽路径到终端、粘贴文件等操作
  • WebGL 加速渲染：流畅的终端显示体验
  • 深色/浅色主题：一键切换，终端配色自动联动

系统要求

项目 要求
操作系统 Windows 10 / 11
运行环境 无需额外安装（已内置 Electron 运行时）
Claude Code 需已安装 Claude Code CLI（用于会话管理和定时任务功能）
磁盘空间 约 200MB

二、安装与启动

安装方式

  1. 双击运行 Split Terminal Setup 1.1.0.exe 安装包
  2. 选择安装目录（可自定义），点击「安装」
  3. 安装完成后，桌面会自动创建 Split Terminal 快捷方式

启动

双击桌面快捷方式或从开始菜单启动 Split Terminal。

启动后，应用会自动检测系统中的 PowerShell 版本：
- 优先使用 PowerShell 7（pwsh）
- 若未安装，回退到 Windows PowerShell（powershell.exe）

三、界面概览

应用界面由以下几个区域组成：

Split Terminal 界面概览示意图

  • 标签栏：管理多个标签页，每个标签页拥有独立的终端面板组
  • 文件浏览器侧边栏：树形文件管理器，可拖拽调整宽度（120~600px）
  • 终端面板区：核心工作区域，最多同时显示 6 个终端
  • 底部状态栏：显示当前状态信息

四、功能详解

4.1 多分屏终端管理

创建终端

  • 点击终端区域右上角的 「+」按钮 新建一个终端面板
  • 快捷键 Ctrl+Shift+N：新建终端并自动输入 claude 命令

面板布局

应用根据当前面板数量自动计算最佳布局：

面板数 布局方式
1 全屏
2 左右各半
3 上 1 下 2 或上 2 下 1
4 2×2 网格
5 上 2 下 3 或上 3 下 2
6 2×3 网格

面板操作

  • 切换焦点：点击面板或使用 Ctrl+1~6 切换到对应面板
  • 关闭面板：点击面板标题栏的关闭按钮，或 Ctrl+Shift+W
  • 重命名面板：Ctrl+Shift+R，为面板设置自定义名称
  • 调整字体大小：Ctrl+滚轮 上下滚动，字体范围 8~32px

文本操作

  • 复制：选中终端文本后按 Ctrl+C（无选中时发送中断信号）
  • 粘贴：Ctrl+V 粘贴剪贴板内容
  • 大段文本粘贴：自动分块写入（256字节/块），避免内容丢失，同时保护 ANSI 转义序列完整性
  • 中文输入：完整支持 IME 中文输入法

标签页管理

  • 新建标签页：Ctrl+Shift+T 或点击标签栏的「+」按钮
  • 切换标签页：点击标签栏中的标签
  • 重命名标签页：Ctrl+Shift+E
  • 关闭标签页：点击标签上的关闭按钮

每个标签页独立管理自己的终端面板组，互不干扰。

4.2 Claude Code 会话管理

打开方式：点击工具栏的 Claude 会话按钮，或按 Ctrl+Shift+L。

Claude 历史会话面板示意图

会话列表

应用会自动扫描 ~/.claude/projects/ 目录下的会话文件，以列表形式展示所有历史 Claude 会话。

  • 智能去重：同一会话链只显示一条记录，ID 使用最新续接的会话（确保进入后看到完整对话内容）
  • 搜索：在搜索框输入关键词，快速过滤会话
  • 排序：按最近使用时间排序，置顶会话始终在最前

会话操作

操作 说明
点击会话 自动创建新终端面板，执行 claude --resume <会话ID> 恢复对话
置顶/取消置顶 点击星标图标，将常用会话固定在列表顶部
重命名 为会话设置自定义名称，方便识别
删除 从列表中移除会话记录

自动关联

当你在终端中手动输入 claude 命令启动新会话时，应用会自动轮询检测新建的 Claude 会话，并将其关联到当前终端面板，方便后续管理。支持 claude --resume <UUID> 方式直接关联已有会话。

会话详情预览

在会话列表中，可以查看每个会话的详情预览，包括最近 20 条对话消息（用户和 Claude 的问答摘要），每条消息最多显示 200 字符，帮助你快速回忆会话内容。

数据存储与管理

会话管理的数据存储在 Claude Code 的项目目录中：

文件 说明
~/.claude/projects/ Claude Code 会话文件根目录，按项目路径分组存储
*.jsonl 每个会话对应一个 JSONL 文件，文件名即会话 ID
_session-names.json 自定义会话名称映射表（每个项目目录下独立存储）
_session-pins.json 置顶会话 ID 列表（每个项目目录下独立存储）

会话列表规则

  • 显示上限：最多显示 50 条会话记录
  • 排序规则：置顶会话优先，然后按最后活跃时间倒序排列
  • 智能去重：基于 slug 分组，同一会话链（原始会话 + 续接会话）只显示一条，使用最新续接会话的 ID（确保恢复后看到完整对话）
  • 项目过滤：如果当前终端有工作目录，会话列表会按项目路径前缀过滤，只显示相关会话

4.3 定时任务调度

打开方式：点击工具栏的定时任务按钮，或按 Ctrl+Shift+J。

定时任务功能允许你设定计划，让 Claude 在指定时间自动执行指令。

创建定时任务

  1. 打开定时任务面板
  2. 点击「新建任务」
  3. 填写任务配置：
配置项 说明
关联会话 选择要恢复的 Claude 历史会话
执行指令 要发送给 Claude 的指令内容
调度方式 每天定时（daily）/ 固定间隔（interval）/ 一次性（once）
完成后操作 可选：关机、锁屏、睡眠、休眠，或不执行任何操作

调度方式详解

  • 每天定时（daily）：设定每天的执行时间（如 09:00），每天自动触发
  • 固定间隔（interval）：设定间隔时间（如每 2 小时），周期性执行
  • 一次性（once）：设定具体日期和时间，仅执行一次

任务执行流程

创建新标签页 → 启动终端 → claude --resume 恢复会话
    → 等待 Claude 就绪 → 发送指令 → 等待执行完成
    → 执行完成后操作（如有）

就绪检测：应用通过三种方式判断 Claude 是否准备好接收指令：
1. 检测到提示符 >
2. 检测到 TUI 输入框
3. 输出稳定 8 秒（保底机制）

完成检测：输出稳定 30 秒或 PTY 进程退出。

任务管理

  • 启用/禁用：切换任务的激活状态
  • 中止运行：对正在执行的任务发送 Ctrl+C 中断信号并终止进程
  • 查看日志：每个任务保留最近 50 条执行日志，记录执行时间和结果
  • 调度频率：系统每 60 秒检查一次是否有任务需要执行

4.4 文件浏览器

打开方式：点击工具栏的文件浏览器按钮，或按 Ctrl+Shift+B 切换显示/隐藏。

基本操作

  • 打开文件夹：点击侧边栏顶部的「打开文件夹」按钮，选择要浏览的目录
  • 展开/折叠：点击文件夹图标展开或折叠子目录
  • 双击文件夹：自动在当前终端执行 cd 命令切换到该目录
  • 拖拽文件：将文件或文件夹拖拽到终端面板，自动插入完整路径

文件管理

通过右键菜单可执行以下操作：

操作 说明
新建文件 在选中目录下创建新文件
新建文件夹 在选中目录下创建新文件夹
重命名 重命名选中的文件或文件夹
删除 删除选中的文件或文件夹
粘贴文件 将剪贴板中的文件粘贴到选中文件夹

粘贴文件

选中目标文件夹后，按 Ctrl+V 可将系统剪贴板中复制的文件粘贴到该文件夹中（支持 Windows 资源管理器中复制的文件）。

最近打开

应用会记住最近打开的 10 个文件夹，启动时自动恢复上次打开的目录。

侧边栏宽度

拖拽侧边栏右边缘可调整宽度，范围 120~600px。

4.5 主题切换

应用提供两种主题：

主题 说明
深色主题（默认） 科技风深色配色，适合长时间使用
浅色主题 明亮配色，适合光线充足的环境

切换方式：按 Ctrl+Shift+K 一键切换。终端颜色方案会自动跟随主题变化。

六、常见问题

Q1：启动后终端显示空白？

检查系统是否安装了 PowerShell。应用优先使用 PowerShell 7（pwsh），若未安装则使用 Windows 自带的 PowerShell。

Q2：Claude 会话列表为空？

确保已安装 Claude Code CLI，并且在 ~/.claude/projects/ 目录下存在会话记录文件。至少需要使用过一次 claude 命令创建过会话。

Q3：定时任务没有按时执行？

  • 确认任务处于「启用」状态
  • 确认关联的 Claude 会话仍然有效
  • 系统每 60 秒检查一次调度，实际执行时间可能有最多 1 分钟的偏差
  • 确保应用保持运行状态（最小化到任务栏即可，不要关闭）

Q4：中文输入法无法正常使用？

v1.1.0 已修复 IME 中文输入兼容问题。如仍有问题，尝试点击终端面板重新获取焦点后再切换输入法。

Q5：粘贴大段文本后内容不完整？

v1.1.0 已优化大段文本粘贴机制，采用分块写入（256字节/块，50ms间隔）。如仍有问题，建议将文本保存为文件后在终端中使用命令读取。

Q6：文件树频繁闪烁？

v1.1.0 已优化文件树刷新策略，仅响应文件新增/删除事件，并采用增量 DOM 更新减少闪烁。
